# 小红书高质量文案创作技能

## 核心原则

**真实 > 营销感**：小红书社区对AI生成内容和营销号语气极度敏感。文案必须像真人写的。

**篇幅控制**（2026-06-29新增）：小红书文案统一为**中等篇幅**。
- 字数：300-600字（正文部分，不含标签）
- 结构：3-4个模块，每模块3-5行
- 原则：说完核心观点就停，不贪多求全
- 压缩技巧：一个观点只配1-2个例子，删掉次要细节和过度展开

## 创作规范

### 1. 语气规则

| ✅ 推荐 | ❌ 禁止 |
|---------|---------|
| 像跟朋友聊天一样自然 | "姐妹们兄弟们" |
| 有观点有态度 | "真的狂喜""太震撼了" |
| 克制使用emoji（全文≤5个） | 每段都堆emoji |
| 短句+长句交替 | AI式排比句 |
| 承认不确定性 | "唯一""最强""颠覆" |

### 2. 事实核查清单

发布前必须逐项确认：

- [ ] **所有数据/奖项有明确来源**：不能仅依赖用户提供的信息，需交叉验证
- [ ] **专业术语准确**：CVPR≠SIGGRAPH，不能混淆
- [ ] **引用可追溯**：附上论文链接、官网链接、新闻源
- [ ] **不确定的内容标注**："目前无法确认""建议以官方为准"
- [ ] **竞品/对比信息公平**：不拉踩，不虚假宣传

### 3. 内容结构模板

```
【钩子】1-2句，说清楚为什么值得看（不用感叹号堆叠）

【正文】分3-5个模块，每模块聚焦一个点
- 用数据/事实支撑观点
- 提供行业视角的专业分析
- 指出不确定性或争议点

【实用建议】给读者可操作的信息
- "建议实际测试XX"
- "重点关注XX指标"

【互动引导】自然地邀请讨论
- "欢迎在评论区交流实际体验"
- 不用"点赞收藏关注"三连
```

### 4. 常见错误避坑

#### 错误1：事实错误（最致命）
```
❌ "CVPR 2025 Best Paper唯一获奖中国团队"
✅ 先查证：CVPR 2025 BP是否真的是这家？方向是否匹配？
✅ 不确定时写："据悉获得CVPR/SIGGRAPH相关奖项（以官方为准）"
```

#### 错误2：AI味太重
```
❌ "这组数据直接颠覆认知！"
❌ "设计师狂喜。真的狂喜。"
❌ "一口气看完了，干货密度极高"
✅ "发布会上几个关键数字：..."
✅ "这个功能如果真能实现，对工作流的影响是..."
```

#### 错误3：缺乏行业深度
```
❌ 只写"4秒生成！80秒生成！"（数字党）
✅ 讨论"生成的模型拓扑是否干净""能否用于rigging"
✅ 区分"可用于3D打印"vs"可用于游戏/影视生产管线"
```

#### 错误4：忽略社区反馈
```
❌ 关闭评论区或无视质疑
✅ 正面回应专业质疑（如"破面"问题）
✅ 在文案中主动提出可能的局限性
```

### 5. 领域适配

不同领域的内容需要不同的专业深度：

| 领域 | 关键关注点 | 专业术语示例 |
|------|-----------|-------------|
| 3D/图形学 | 拓扑、布线、UV、rigging | 四边面、水密性、CoWoS |
| AI/大模型 | 训练数据、参数量、推理延迟 | Token、LoRA、RAG |
| 半导体 | 制程、良率、国产化率 | nm、EUV、CMP |
| 消费电子 | 实际体验、对比测试 | 续航、发热、色彩还原 |

### 6. 发布前自检

发布前问自己3个问题：

1. **业内人士看到会觉得外行吗？** → 如果会，补充专业细节或标注不确定性
2. **一眼能看出是AI写的吗？** → 如果能，修改语气和句式
3. **有事实错误被锤的风险吗？** → 如果有，核实后再发

### 7. 流量优化（2026-06-30新增）

基于真实流量表现反推的优化规则：

**标签策略**
- ❌ 标签粘连（`#人工智能#互联网`）→ 平台无法解析
- ❌ 重复标签 → 算法降权
- ❌ 无关标签（科技帖打`#日常文案`）→ 精准度扣分
- ❌ 太泛标签（`#AI`）→ 竞争极端激烈，新帖无曝光
- ✅ 5个精准长尾标签（如`#AI工具` `#效率工具` `#AI编程`）
- ✅ 标签间用空格隔开

**时间线敏感度**
- ❌ 在2026年引用"2023年ChatGPT刚出来" → 读者判定为旧文洗稿
- ✅ 用"现在""最近""当前"等即时词汇

**格式适配**
- ❌ 学术论文格式（一、二、三、四）→ 小红书算法判为AI生成
- ❌ 教科书条目式（RPA阶段、Copilot阶段...）→ 用户0.5秒划走
- ✅ 自然叙事流，像朋友在讲一件事

**钩子法则**
- 前20字决定80%点击率
- ❌ 平淡开头 → "最近整理了一下这个方向"
- ✅ 冲突/悬念/利益开头 → "刷到一条帖子说..."或"这件事背后有个被忽略的逻辑"

**个人体验是刚需**
- ❌ 纯知识转述 → 没有人味，没有差异化
- ✅ 必须包含：我用过XX、我踩过的坑、我的实际感受
- 原则：你先出牌（分享自己的体验），读者才跟牌（评论互动）

**账号权重认知**
- 新账号/低粉账号首帖 → 平台小流量测试
- 点击率低+完读率低+互动率低 → 直接断流
- 前3-5篇是关键，宁可少发也要高质量

## 快速参考

```
标题：有信息量，不做标题党
开头：直接说事实，不用"姐妹们"
正文：有数据，有来源，有观点
结尾：邀请讨论，不求三连
标签：3-5个相关标签，不堆砌
```

### 8. 成功模式 vs 失败模式（2026-06-30 基于真实数据新增）

#### 内容类型判断（发之前先归类）

| 类型 | 定义 | 适合小红书吗 |
|------|------|------------|
| 技术策展 | "我发现了一个有意思的项目/趋势" | ✅ 最合适 |
| 个人经验 | "我是怎么解决这个问题的" | ✅ 很合适 |
| 知识搬运 | "XX是什么，分为几个阶段" | ⚠️ 容易像教科书 |
| B2B软文 | "这个产品功能多牛逼" | ❌ 平台限流 |
| 甲方PR稿 | "参加了XX发布会，震撼了" | ❌ 一眼广告 |

#### 成功笔记的9个特征

1. **开头钩子**：传递时效性+价值感（"忍不住立刻整理出来"）
2. **明确信源**：有具体人名/团队背景，不是匿名转述
3. **核心论点**：一句话说清观点，观点驱动>产品驱动
4. **结构像观点不像教科书**：emoji分节+问题式标题，不用"一、二、三"
5. **可操作性**：读者看完能立刻行动（"去GitHub下载"）
6. **技术深度刚好**：内行不嫌浅，外行不嫌难
7. **有人味儿**："诚意拉满""打在设计痛点上"——真人口语
8. **标签精准**：借势大词（如`#Figma替代`）+ 观点标签（如`#审美即能力`）
9. **结尾有收束感**：核心观点收尾，不是空泛的"欢迎评论区聊聊"

#### 失败笔记的8个死穴（对照自查）

1. 标题和内容断裂（"只做了一个决定"→点进去是产品功能列表）
2. 产品宣传口吻（"它是你的XX"排比式功能介绍）
3. 情感表达过度虚假（"起鸡皮疙瘩""被暴击""印钞增长曲线"）
4. 功能列表式正文（零可读性，没人会读产品spec）
5. 品牌标签暴露营销意图
6. 引用发布会PPT数据而非个人实测
7. 变现意图外露（"相当于白捡2-3个业务员"）
8. 过时时间线（2026年用"2023年ChatGPT刚出来"开头）

#### 核心法则

**你是读者的信息策展人，不是产品的推销员。**
小红书算法+用户直觉对营销内容的识别已经非常成熟。好的技术帖让读者觉得"这人帮我筛选了一个好东西"，而不是"这人想卖我东西"。
